# astro-blog

An edge-native, SEO-first personal blog built on **Astro 6** and deployed to
**Cloudflare Workers** (Static Assets). Every page is prerendered at build
time and served from 320+ Cloudflare data centers — zero JS shipped by
default, zero cold-start, zero origin.

## Why this stack

| Concern     | Choice                             | Why                                                                |
| ----------- | ---------------------------------- | ------------------------------------------------------------------ |
| Framework   | Astro 6 + content collections      | Zero-JS by default, typed frontmatter, MDX, fastest SSG            |
| Hosting     | Cloudflare Workers (Static Assets) | Sub-50 ms TTFB globally, no cold-start, room to grow into SSR      |
| Styling     | Tailwind CSS v4 (PostCSS)          | CSS-first config, dark mode, no runtime                            |
| i18n        | Astro built-in (tr + en)           | Locale-prefixed routes, per-locale RSS, hreflang via sitemap       |
| SEO         | Sitemap + RSS + JSON-LD + OG/TC    | Every signal Google + social platforms expect, baked into BaseHead |
| Type safety | Astro + Zod schemas + strict TS    | Frontmatter, props, and SEO inputs all type-checked                |

## Quick start

## Project layout

```
src/
├── components/      Reusable UI (BaseHead, Header, Footer, FeaturedCard, PostRow, Tag, ThemeToggle)
├── content/blog/    Markdown + MDX posts under tr/ and en/ subdirs
├── content.config.ts  Zod schema for the blog collection
├── i18n/            Per-locale UI strings (tr, en) and helpers
├── layouts/         BaseLayout (everything) + PostLayout (article pages)
├── lib/             format.ts (dates, reading time), posts.ts (queries)
├── pages/           File-based routing — every page has `export const prerender = true`
│   ├── 404.astro
│   └── [lang]/      Locale-prefixed routes: /tr/*, /en/*
├── styles/global.css  Tailwind v4 + dark mode + prose
└── consts.ts        Site-wide config (url, author, social, nav) — edit this first
public/              Static assets (favicon, OG default, avatar, …)
astro.config.mjs     Astro config — sitemap + MDX, i18n (default tr)
wrangler.jsonc       Cloudflare Workers deployment config (Static Assets)
```

## Writing a post

Drop a Markdown or MDX file under `src/content/blog/<lang>/` where `<lang>`
is `tr` or `en`. Frontmatter is type-checked against `src/content.config.ts`:

```md
---
title: "A new post"
description: "Short summary used in cards, RSS, and OG/Twitter previews."
pubDate: 2026-05-20
tags: ["edge", "performance"]
featured: false                # surfaces on the homepage
draft: false                   # drafts only render in `npm run dev`
translationKey: "hello-world"  # optional; links tr/en variants of the same post
---

Your content here.
```

The slug comes from the file name (e.g. `hello-world.md` under `en/` →
`/en/blog/hello-world`).

## SEO features that ship out of the box

- **Per-page metadata** — title, description, canonical, robots, theme-color
- **Open Graph + Twitter Cards** — including `article:*` fields for posts
- **JSON-LD `@graph`** — `WebSite`, `Person`, `BreadcrumbList`, `BlogPosting` on posts
- **Per-locale RSS** — `/tr/rss.xml` and `/en/rss.xml` (full RSS 2.0)
- **`sitemap-index.xml`** — generated by `@astrojs/sitemap`, with hreflang for tr ↔ en
- **View transitions** — instant-feeling navigation, with reduced-motion respect
- **Dual-theme syntax highlighting** — Shiki light/dark synced with the site theme

## Deploying to Cloudflare

1. `npx wrangler login`
2. Update `wrangler.jsonc`:
   - Set `name` to your Worker name (becomes `<name>.<account>.workers.dev`)
   - Add a `routes` block or custom domain when you have one
3. Update `src/consts.ts` — at minimum the `url`, `title`, and `author` fields
4. `npm run deploy`

Builds are reproducible — the same git SHA always produces the same `dist/`.

### Adding bindings (KV, D1, R2, …) or server endpoints

The current setup is **assets-only** — no Worker code, just Cloudflare
serving the prerendered HTML/CSS/JS. To add server logic:

1. `npm install @astrojs/cloudflare`
2. Re-add the adapter to `astro.config.mjs`:
   ```js
   import cloudflare from "@astrojs/cloudflare";
   adapter: cloudflare({ imageService: "compile" }),
   output: "server",
   ```
3. Add `"main": "./node_modules/@astrojs/cloudflare/dist/entrypoints/server.js"` to `wrangler.jsonc`
4. Add your binding to `wrangler.jsonc` and run `npm run cf-typegen`
5. Access via `Astro.locals.runtime.env.MY_BINDING` in any page or endpoint
6. Keep `export const prerender = true` on every page that doesn't need SSR

## Customising

- **Brand colours** — change `--color-brand-*` in `src/styles/global.css` and `SITE.themeColor` in `src/consts.ts`
- **Navigation & social** — edit `NAV_KEYS` / `NAV_PATHS` and `SITE.social` in `src/consts.ts`
- **Adding a locale** — extend `locales` in `astro.config.mjs`, add `src/i18n/<lang>.ts`, mirror posts under `src/content/blog/<lang>/`
- **Default OG image** — replace `public/og-default.svg`
- **Favicon** — replace `public/favicon.svg`
